# Вариант 1 

Реализовать gRPC-сервис ProductCatalog с методом:

GetProductDetails(ProductID)

Метод должен возвращать информацию о товаре по его идентификатору.
Тип взаимодействия — Unary RPC (один запрос → один ответ).

# 🏗 Архитектура решения

В работе реализована классическая клиент-серверная архитектура.
```
+-------------+        gRPC         +-------------+
|   Клиент    |  <--------------->  |   Сервер    |
|  client.py  |                     |  server.py  |
+-------------+                     +-------------+
                                          |
                                          |
                                   +---------------+
                                   | products_db   |
                                   |  (в памяти)   |
                                   +---------------+
```
# Компоненты системы

Client (client.py)
Отправляет запрос серверу и получает информацию о товаре.

Server (server.py)
Обрабатывает запрос клиента и возвращает данные о товаре.

product.proto
Контракт взаимодействия между клиентом и сервером.
Описывает метод сервиса и структуры сообщений.

products_db
Имитация базы данных (50 товаров, хранящихся в оперативной памяти).

# 🧩 Описание реализации

В рамках лабораторной работы был создан gRPC-сервис ProductCatalog.

Описание сервиса выполнено в файле:
```
product.proto
```
Сервис содержит метод:

GetProductDetails — возвращает информацию о товаре по его ID.

Метод реализован как Unary RPC, что означает:

клиент отправляет один запрос;

сервер возвращает один ответ.

(Простыми словами: один вопрос — один ответ.)

📷 Содержимое файла product.proto
```
syntax = "proto3";

package product;

service ProductCatalog {
  rpc GetProductDetails (ProductID) returns (ProductResponse) {}
}

message ProductID {
  int32 id = 1;
}

message ProductResponse {
  int32 id = 1;
  string name = 2;
  string description = 3;
  float price = 4;
  bool in_stock = 5;
}
```
⚙ Генерация gRPC-кода

После создания product.proto была выполнена команда:
```
python -m grpc_tools.protoc -I. --python_out=. --grpc_python_out=. product.proto
```
В результате автоматически были созданы файлы:
```
product_pb2.py

product_pb2_grpc.py
```
Эти файлы обеспечивают взаимодействие клиента и сервера через gRPC.

🖥 Реализация сервера

Серверная часть реализована в файле:

server.py

Создан класс ProductCatalogService, наследующийся от
ProductCatalogServicer.

В сервере реализован метод:

GetProductDetails — выполняет поиск товара по ID и возвращает информацию о нём.

В системе реализована генерация 50 товарных позиций, которые хранятся в памяти (имитация базы данных).

Сервер запускается на порту:
```
50052
```
и ожидает подключения клиентов.

💻 Реализация клиента

Клиентская часть реализована в файле:
```
client.py
```
Подключение выполняется через:
```
grpc.insecure_channel('localhost:50052')
```
Клиент позволяет:

ввести ID товара;

отправить запрос серверу;

получить и вывести информацию о товаре.

Вызов метода выполняется через:
```
response = stub.GetProductDetails(request)
```

📊 Результат работы

Сервис корректно:

принимает ID товара от клиента;

возвращает информацию о товаре (Unary RPC);

обрабатывает до 50 различных товарных позиций;

возвращает сообщение "Not found" при отсутствии товара.
